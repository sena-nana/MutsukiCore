# v0.1 版本报告 —— 最小可运行骨架

## 方向

第一个可装载、可运行、可被测试的 Agent 闭环。设计基调：参考 NoneBot 心智、完整类型、通过元类 + 虚拟类（ABC/Protocol）+ 注册表简化插件语法，同时严守 MutsukiCore 的 Agent 一等公民、领域中立、双协议分离等所有 hard rules。

## 完成项

### contracts 层（[mutsukicore/contracts/](../../mutsukicore/contracts/)）

- `Contract` 基类（`msgspec.Struct`，`__init_subclass__` 自动注册到 `SchemaRegistry`）
- 类型化 ID：`AgentId / TraceId / SpanId / RefId / MessageId`（`NewType`）
- `LifecyclePhase`、`Capability`、`Message` / `ContentPart` / `ChannelRef`、`Event` / `TraceSpan`、`Decision`
- `Error` + `RecoveryAction` + `ErrorCode` 注册式 + `Errs` 内置门面（14 个错误码）
- `Handle[T]` 抽象虚拟类、`RefPayload[T]`、`RefDescriptor`、`BackpressureChannel[T]`、`Replayability`
- `PluginManifest` / `CommandSpec` / `ContractDep` / `PluginDep` / `ServiceDep` / `Arg` / `Inject` / `RefArg`
- `Service` Protocol、`ServiceMode` 枚举
- **`CapabilityName(str)` 注册式 + `Caps` 门面**（READ_MESSAGE / SEND_MESSAGE / CALL_LLM / PERSIST / NETWORK_EGRESS / SPAWN_AGENT / HOLD_REF / BORROW_REF / PRODUCE_REF_STREAM）
- **`PermissionName(str)` 注册式 + `Perms` 门面 + `PermissionRule`（`&` / `|` 组合）**
- `register_schema_compatibility` API（默认 byte-equal 兼容回调）—— **API-only**：v0.1 仅落 API + 默认策略 + 单元测试（[tests/contracts/test_schema_compat.py](../../tests/contracts/test_schema_compat.py)），codec / bus 层的实际拦截调用推迟到 v0.2

### core 层（[mutsukicore/core/](../../mutsukicore/core/)）

- **`PluginMeta` 元类 + `Plugin(ABC, Generic[C])` 虚拟类** —— 定义时校验 ClassVar `id`/`version`/`capabilities`、嵌套 `Config(msgspec.Struct)`，扫描 `@command`-标记方法，构造 `PluginManifest`，注册到 `PluginRegistry`，提供精确 `__repr__`（含来源 file:line）
- **`@command` 装饰器** —— 用 `inspect.getdoc` + `docstring_parser` 解析 docstring（首段=description；`Args:` 段=参数 desc），合并 `Annotated[T, Arg(...)]` 约束，合成 JSON Schema
- **`Dependent[R]` + `Param` ABC**（NoneBot 心智，去掉按名 fallback）—— `CtxParam` / `ArgParam` / `ServiceParam` / `RefParam`
- `PluginScope` / `TransactionScope` —— 资源生命周期追踪 + 反向清理 + Handle 泄漏检测
- `RefCountedHandle` —— 通用 `Handle[T]` 实现 + `make_stub_handle()` 测试工厂
- `Bus` —— direct-dispatch 快路径 + 默认并发派发
- `Saga` —— 多步事务 + 自动反向补偿
- `Lifespan` —— `on_spawn / on_awake / on_sleep / on_stop` 钩子列表
- `ServiceContainer` —— by-value / by-ref 解析
- `check_capabilities` —— Capability 静态准入
- `PluginLoader` —— `entry_points` 扫描 + DAG 拓扑 + 加载/卸载，含 Kahn's algorithm 环检测
- `Agent` —— 身份、Context、生命周期、inbox/outbox、加载的插件实例集合
- `AgentContext` + `TraceContext` 工厂

### runtime 层（[mutsukicore/runtime/](../../mutsukicore/runtime/)）

- `Clock` Protocol + `SystemClock` + `ManualClock`（测试用，可控时间推进）
- `IdGen` Protocol + `NanoIdGen` + `DeterministicIdGen`（决定性回放）
- `RNG` Protocol + `SeededRng`
- `AgentScheduler` —— v0.1 最小 tick 循环：inbox → 命令路由 → Capability 检查 → Permission 检查 → Dependent.solve → outbox + TraceSpan
- `loop` —— asyncio façade。**同步点检测在 v0.1 仅有占位钩子 `install_sync_point_guard`**：v0.1 阶段对 `time.sleep` / `uuid.uuid4` / `random` 全局源的拦截依赖 code review + ruff `select` 中的 ASYNC 规则（间接覆盖），运行时守卫（`sys.settrace` 或 ruff 自定义规则）推迟到 v0.2

### adapters 层（[mutsukicore/adapters/](../../mutsukicore/adapters/)）

- `Adapter` ABC + `AdapterCapability` 枚举（自动 `__init_subclass__` 注册）
- `InMemoryAdapter` —— 测试驱动入口：`send_text` / `drain_outbox`

### plugins 层

- `EchoPlugin`（[mutsukicore/plugins/echo/](../../mutsukicore/plugins/echo/)）—— v0.1 参考插件 + `echo.smoke` 入口
- 通过 `pyproject.toml` `[project.entry-points."mutsukicore.plugins"]` 注册，与第三方插件走同一加载路径

### observability 层

- `JsonlTraceWriter` —— 订阅 `trace.span` 事件，写 JSONL（旁路，不被任何层依赖）

### tests/

61 个测试，覆盖：契约（capability/permission 注册、refpayload 生命周期 + 泛型注册不污染、core 无领域字样、schema-compat API）、core（PluginMeta 校验 + Dependent 缓存、Dependent typed-DI、Scope 清理与泄漏、DAG 拓扑、Capability 守卫、Agent fallback scope 复用）、runtime（决定性时钟与 IdGen、ManualClock heap 推进 + waiter 阈值告警 + cancel_all、AgentScheduler 错误码分类 + stop 异常透传）、plugin（echo 全生命周期、命令+工具 schema 共用、热重载 100 次无泄漏）。

## v0.2 候选门槛

- **真实 adapter**：CLI（typer）+ 至少一个 IM 平台 SDK 层（OneBot v11 或 WS）。
- **配置 schema 自动校验**：YAML/TOML 加载 → msgspec.Struct 校验 → 装载前拒绝。
- **同步点检测**：v0.1 在 `runtime/loop.py` 留了 hook，v0.2 用 `sys.settrace` 或 ruff 自定义规则覆盖 `time.sleep` / `uuid.uuid4` / `random` 全局源在插件包内的调用。
- **`make_plugin_meta(other_meta)`** 多元类合成工具（仅在出现需求时引入）。
- **Yume `StimulusEvent → ExpressionDecision` 样本** 用承载性测试套件做 msgspec roundtrip。
- **Schema 兼容回调测试用例** 覆盖 `register_schema_compatibility` 的版本协商行为。
