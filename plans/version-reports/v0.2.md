# MutsukiBot v0.2 版本报告

## 方向

v0.2 的目标是把 MutsukiBot 从“可运行骨架”推进到“通用 Agent 框架”：删除旧 Adapter 抽象，统一为 Plugin + Operation/Source/Dispatcher，并用 transport plugin 形态承接外部协议接入。

## 完成项

- `Dispatcher` 完成 Operation / Source 注册、invoke、publish 广播与状态查询。
- `AgentRegistry` 完成多 Agent 同进程广播。
- `PluginLoader` 支持 raw mapping 配置并在装载时做 `msgspec.convert` 校验。
- `InMemoryEndpointPlugin` 作为测试 / 冒烟 transport reference plugin。
- `TodoPlugin` 作为 tool endpoint 范例，验证跨 endpoint 调用。
- `QqToTodoPlugin` 验证跨 endpoint 协作链路。
- `OneBotV11Plugin` 作为首个真实 IM transport reference plugin，采用反向 WebSocket。
- hard rule #14 lint、dispatcher contract helper、dispatcher invoke benchmark 基线补齐。
- docs 从 adapter 语义迁移到 transport plugin / endpoint / dispatcher 语义。

## 效果检查

- Agent 仍然是一等运行时实体。
- core 与 contracts 不含 OneBot 专有协议类型。
- transport 资源通过 `Handle` + `PluginScope` 托管。
- dispatcher 反注册、卸载与广播行为均有回归覆盖。
- 文档与实现已对齐到 v0.2。

## 下一版门槛

- v0.3 重点是 ResourceHost、跨 agent invoke、优先级 / 选举、资源协商与 saga 扩展。
- 若后续 transport 继续扩展，仍保持 plugin 形态，不恢复独立 Adapter 抽象。
