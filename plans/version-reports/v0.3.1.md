# Mutsuki v0.3.1 阶段报告

## 方向

v0.3.1 承接 v0.3 MVP 的后续门槛，补齐类型化 `Handle` 注入、`ResourceHost`
策略化资源维护，以及 dispatcher / ResourceHost 的最小 trace 因果链。

## 完成项

- `RefArg` 增加 `source` / `ref_id` / `host_name`，支持 payload 与
  `ResourceHost` 两条来源。
- `Dependent.RefParam` 只认领 `Annotated[Handle[T], RefArg(...)]`，并校验
  `RefDescriptor.kind`。
- `ResourceHost` 增加托管句柄索引、`ResourceRecord`、`get_handle` /
  `get_handle_for`、可插拔 eviction / keepalive policy。
- `ResourceHost.acquire_for` / `release_for` / `get_handle_for` 发出 trace span。
- `dispatch.invoke` / `dispatch.invoke_in_agent` 发出 trace span；跨 Agent 调用中
  目标 Agent 的 operation span 继承调用方 span。
- 新增结构化错误码：`ref.not_found` 与 `ref.kind_mismatch`。

## 下一步门槛

- 选举策略插件化：把当前 priority + agent_id 的固定选择提升为可替换策略。
- trace 回放闭环：让 `TraceSpan` 写入与回放测试 kit 形成可复用契约。
- ResourceHost 策略参数治理：为 eviction / keepalive policy 增加配置 schema 与
  contract tests。
