# MutsukiCore v0.3.4 阶段报告

## 方向

v0.3.4 承接 v0.3.1~v0.3.3 已完成的 ResourceHost / trace / election 底座，
把 ResourceHost 的 eviction / keepalive 从“可传任意函数”收束成“可声明、
可校验、可测试”的治理配置。

## 完成项

- 新增 `ResourceRecordSelector` 契约，作为资源策略的通用配置 schema。
- 新增 `ResourceHostPolicyConfig` 契约，分别承载 eviction / keepalive 策略配置。
- `ResourceHost` 支持直接接收 `policy_config`，并把 selector 编译成谓词。
- `ResourceHost` 继续兼容显式 callable 策略，但同一策略不能同时由 config
  与 callable 提供。
- 策略配置校验会拒绝未知字段、空 selector、明显冲突的 prefix / exact 组合。
- 新增 contract tests 与 core tests，覆盖 schema 注册、selector 匹配、未知键拒绝
  与配置/调用冲突。
