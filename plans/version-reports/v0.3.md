# Mutsuki v0.3 阶段报告

## 方向

v0.3 从 v0.2 的 Operation / Source / Dispatcher 骨架继续向多 Agent 协作和资源生命周期推进。MVP 只落地可测试的最小内核：跨 Agent invoke、确定性目标选择、ResourceHost、资源租约和 Saga 结构化错误。

## 目标检查

- 跨 Agent 调用必须显式指定目标 Agent，不隐式广播。
- ResourceHost 只管理进程内资源；跨进程和持久化资源后续再做。
- 资源协商只做容量计数，不做调度队列。
- 选举只做确定性排序，不引入复杂策略系统。

## 下一版门槛

- 类型化 Handle 注入接入 `Dependent`，用 `Annotated[Handle[T], RefArg(...)]` 从 ResourceHost 或调用 payload 安全解析。
- ResourceHost 支持可插拔 eviction / keepalive 策略。
- 跨插件 trace 串起 `dispatch.invoke_in_agent` 与 ResourceHost acquire/release。
